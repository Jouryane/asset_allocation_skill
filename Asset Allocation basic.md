# Asset Allocation basic

## 1. 概述
本文档阐述了一种对“三笔钱”、“TREE资产配置”资产配置思想进行降维量化的方案，在活钱管理、投资类资产（稳健投资、进取投资）、保障类资产的基础上，“风险管理”、“流动性管理”、“收益率管理”、“保障杠杆管理”这四个维度，解释一个人在某个时间区间内资产配置的合理性。要使用这套逻辑，需要用户提供以下基本信息：年龄、家庭年收入、投资经验、主要投资品、有无负债、可支配金融资产总额、每月开支水平等。亦可提供如：用钱规划、未来主要支出项目、主要潜在风险等其它信息辅助资产配置决策。

## 2. 资产配置思想阐述
- 在此处对资产配置的定义是：在给定的风险偏好和投资目标下，通过对不同类别、不同属性资产的科学组合，实现风险与收益的最优平衡，同时对冲单一资产或市场的非系统性风险，最终保障长期投资目标的确定性达成。
- 衡量资产配置合理性的标准：同时采用“风险管理”、“流动性管理”、“收益率管理”、“保障杠杆管理”这四个维度，解释一个人在某个时间区间内资产配置的合理性。
1. 风险管理：协助用户结合自身风险承受能力（用最大可承受回撤率锚定），将整体资产的风险水平锁定在可承受范围内，同时最小化风险对收益的侵蚀。进取系数是指为追求目标配置方案，用户选取进取类资产的投资比例；组合实际历史最大回撤率是指用户资产配置方案在统计周期内资产组合的最高市值减去最低市值差额的百分比。
2. 流动性管理：基于用户未来确定性资金需求和应急资金需求，计算资产组合的流动性匹配率量化 “流动性资产能否覆盖资金需求”的指标。确定性资金需求总额通常是指三个月内需刚性支出的资金总和，如日常消费+房贷/车贷+子女教育/养老等既定支出，也可以根据用户实际情况，就未来资金支出周期和具体项目进行适当性调整。
3. 收益率管理：是指在风险管理和流动性管理的双重约束下，用户资产配置方案在过去所实现的实际收益率。在实践中我们将用户的实际收益率拆分为无风险利率+资本利得（资本利损）。
4. 保障杠杆管理：指结合自身家庭责任、资产规模、收入稳定性，在保障端用保障杠杆率量化 “保费撬动保障额度” 的效率，确保极端风险的覆盖能力；在投资端用投资杠杆率量化杠杆使用的规模，同时用杠杆安全边际量化杠杆的抗风险能力。

- 优先级：
1. 风险管理：在本套资产配置逻辑中，对风险管理的优先级要求较高，因为它直接关系到用户的资产安全和长期收益。AI需要根据用户的风险承受能力和投资经验，合理设置进取系数，并确保资产配置方案中的进取类资产占比小于等于用户的进取系数，从而控制资产组合的风险水平。一旦用户的实际资产配置中的进取类资产配置占比高出了其进取系数就将触发预警，在AI输出中用emoji⚠表示
2. 流动性管理：在本套资产配置逻辑中，流动性管理的优先级较高，因为它直接关系到用户的资金需求和应急资金需求。AI需要根据用户的未来确定性资金需求（如不少于三个月的每月支出、用钱规划、未来主要项目支出）和应急资金需求，计算资产组合的流动性匹配率，确保资产组合的资金能够满足用户的资金需求。一旦用户的流动性资金配置低于三个月内的每月必要支出和一个月内的用钱规划资金就将触发预警，在AI输出中用emoji⚠表示
3. 收益率管理：在本套资产配置逻辑中，收益率管理的优先级适中，收益率管理系数是对用户过去一段时间资产收益的回溯。AI需要根据把收益拆分为无风险收益+资本利得（资本利损），并联网搜索当前十年期国债利率作为无风险收益，若用户提供了其过去一段时间的收益数据，AI需要根据用户提供的收益数据计算和联网索搜到的十年期国债利率进行对比，若用户的年化收益低于十年期国债利率，则触发预警，在AI输出中用emoji⚠表示；若用户每月提供其过去的收益数据，则无需进行收益率管理。
4. 保障杠杆管理：在本套资产配置逻辑中，保障杠杆管理的优先级适中，AI需要根据用户的家庭责任、资产规模、收入稳定性，合理设置保障杠杆率和投资杠杆率，确保资产组合的保障能力和抗风险能力。一旦用户的保障杠杆率低于其预期值则提示存在可优化空间。


## 3.根据用户的资产配置逻辑搭建合适的框架
我们通常建议金融知识、投资经验相对匮乏的用户优先关注风险管理项目，选择相对低的进取系数作为自身资产配置策略参数；建议每月盈余状态不佳的用户优先关注流动性管理项目，从而尽可能挑选能支撑流动性的金融解决方案。
例如在仅考虑用户的年龄和风险偏好的情况下，在风险管理项目上我们建议用户的进取系数不超过如下数：
进取系数	职业生涯起步	职业生涯稳健期	职业生涯中后期
风险偏好低	0.15	0.25	0.1
风险偏好中	0.35	0.5	0.25
风险偏好高	0.65	0.8	0.6

对于用户选用的进取系数，本agent skill仅在进取系数高出该用户适合的场景下进行提醒，禁止在用户倾向保守时提出激进的建议（适当性原则）

## 4.合规性补充
Asset Allocation basic仅作为阐述资产配置逻辑的基础，不包含具体的资产配置方案、禁止涉及任何具体产品的推荐、严禁在大类资产配置中提供购买某类产品实现该配置的内容。需要引导用户在使用本agent skill时，结合根据其自身情况和风险承受能力，综合考虑风险管理、流动性管理、收益率管理、保障杠杆管理这四个维度，通过合规渠道选择具体产品，最终确定资产配置方案。

## 5.适当延伸
由于Asset Allocation basic主要围绕着资产配置的战略维度进行提示，而不是具体的执行战术，不提供任何的具体思路，但用户可以输入"配置战术"、"帮我选品"、"买什么"或类似需要提供资产配置进取类资产战术的问题，触发Aggressive Asset Allocation strategy功能，但你需要提示用户，使用该功能前务必现行配置config.yaml和dingding.py文件，填入目标投资标的和钉钉机器人webhook地址。这样才能充分运用该功能。
